# Installation

Pick the SDK for your stack. All three use the same project API key and ingest
endpoint (`POST /v1/logs`).

## Requirements

| SDK | Runtime |
| --- | --- |
| **JavaScript** (`logstack-js`) | Node.js 16+, modern browsers, Deno, Bun |
| **Go** (`logstack-go-sdk`) | Go 1.21+ |
| **Python** (`logstack-py`) | Python 3.7+ |

## Package install

<Tabs items={["JavaScript", "Go", "Python"]}>
  <Tab value="JavaScript">

```bash
npm install logstack-js
# yarn add logstack-js
# pnpm add logstack-js
# bun add logstack-js
```

  </Tab>
  <Tab value="Go">

```bash
go get github.com/mosesedem/logstack/packages/logstack-go-sdk@v1.0.3
```

Go modules in the monorepo use tags shaped like
`packages/logstack-go-sdk/vX.Y.Z`.

  </Tab>
  <Tab value="Python">

```bash
pip install logstack-py

# Optional framework extras
pip install "logstack-py[django]"
pip install "logstack-py[fastapi]"
```

Import name is `logstack` (e.g. `from logstack import LogStackClient`).

  </Tab>
</Tabs>

## Verify installation

<Tabs items={["JavaScript", "Go", "Python"]}>
  <Tab value="JavaScript">

```typescript
import { createLogStack } from "logstack-js";

const logstack = createLogStack({
  apiKey: process.env.LOGSTACK_API_KEY!,
  endpoint: "https://api.logstack.tech",
});

logstack.info("Installation test");
await logstack.flush();
console.log("logstack-js ready");
```

  </Tab>
  <Tab value="Go">

```go
package main

import (
	"context"
	"fmt"
	"os"

	logstack "github.com/mosesedem/logstack/packages/logstack-go-sdk"
)

func main() {
	client := logstack.NewClient(logstack.Config{
		APIKey: os.Getenv("LOGSTACK_API_KEY"),
	})
	defer client.Close()

	_ = client.Info(context.Background(), "Installation test")
	fmt.Println("logstack-go-sdk", logstack.Version)
}
```

  </Tab>
  <Tab value="Python">

```python
from logstack import LogStackClient
import logstack

client = LogStackClient(api_key="ls_test_verify")
client.info("Installation test")
client.close()
print("logstack-py", logstack.__version__)
```

  </Tab>
</Tabs>

## Self-hosted endpoint

Point any SDK at your own API host (no `/v1` suffix required):

<Tabs items={["JavaScript", "Go", "Python"]}>
  <Tab value="JavaScript">

```typescript
const logstack = createLogStack({
  apiKey: process.env.LOGSTACK_API_KEY!,
  endpoint: "https://logs.your-domain.com", // or http://localhost:8080
});
```

  </Tab>
  <Tab value="Go">

```go
client := logstack.NewClient(logstack.Config{
	APIKey: os.Getenv("LOGSTACK_API_KEY"),
	APIURL: "https://logs.your-domain.com",
})
```

  </Tab>
  <Tab value="Python">

```python
client = LogStackClient(
    api_key=os.environ["LOGSTACK_API_KEY"],
    api_url="https://logs.your-domain.com",
)
```

  </Tab>
</Tabs>

<Callout type="info">
  Self-hosted instances use the same SDKs — only the host URL changes.
</Callout>

## CDN (browser, JS only)

```html
<script type="module">
  import { createLogStack } from "https://esm.sh/logstack-js";

  const logstack = createLogStack({ apiKey: "ls_live_xxx" });
  logstack.info("Page loaded", { url: window.location.href });
</script>
```

<Callout type="warn">
  Never expose production API keys in public client-side code. Prefer a backend
  proxy for browser logging.
</Callout>
